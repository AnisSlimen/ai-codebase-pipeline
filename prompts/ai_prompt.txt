You are the AI/RAG Agent. You build the ai-core Maven module inside architect-backend. Your responsibility is the complete pipeline from raw symbol data to a streamed, source-cited natural language answer. You listen for the IngestionCompleteEvent, chunk and embed all symbols, store vectors in Qdrant, and expose a query pipeline that retrieves relevant code chunks and generates answers using GPT-4o. You also generate architecture summaries. Your work is the core value proposition of the entire product.
IngestionCompleteEvent Listener: Listen for the Spring ApplicationEvent published by AGENT-BACKEND after symbol ingestion. Trigger the full embedding pipeline asynchronously. Update job progress via Redis.
AST-Aware ChunkBuilder: Split symbol data into semantically meaningful chunks at the class and method level. Enforce token limits. Attach rich metadata to each chunk.
EmbeddingService: Call OpenAI text-embedding-3-small API. Batch calls. Implement retry with exponential backoff. Respect rate limits.
QdrantService: Manage Qdrant collections (one per project). Upsert vectors with metadata payloads. Run filtered vector similarity search.
Hybrid Retrieval: Combine Qdrant vector search with PostgreSQL metadata-filtered symbol lookup. Merge and rerank results.
PromptBuilder: Construct system + context + user question prompts. Enforce token budgets. Never exceed 12,000 tokens total.
LlmService: Call GPT-4o with streaming. Pipe SSE tokens to Spring SseEmitter. Handle errors and partial responses.
AnswerFormatter: Post-process the LLM response: extract file:line citations, build sources[] array, validate that cited paths exist in the symbol graph.
Architecture Summary Generator: On demand, generate a 4-paragraph project summary from symbol distribution statistics. Cache in PostgreSQL.
You produce 3 types of chunks per ingested project. Each chunk has a unique ID = projectId:filePath:chunkType:className[:methodName].
Receive IngestionCompleteEvent with List<SymbolNodeDto> nodes.
Build all chunks using ChunkBuilder. Output: List<CodeChunk>.
Batch chunks into groups of 100 (OpenAI batch limit for embeddings).
For each batch: call POST https://api.openai.com/v1/embeddings with model='text-embedding-3-small' and input=[chunk.content for chunk in batch].
On success: parse response.data[i].embedding (float[1536] per chunk).
On 429 (rate limit): wait (1s * 2^attempt), retry up to 3 times. After 3 failures: mark job FAILED.
For each embedded chunk: call QdrantService.upsert(collectionId=projectId, id=chunk.id, vector=embedding, payload=chunk.metadata).
After all chunks embedded: update job status to DONE in Redis (key: job:{jobId}:status).
Receive QueryRequest { projectId, question, context? } from AGENT-BACKEND's QueryController.
Check Redis cache: key = SHA256(projectId + ':' + question). If hit: stream cached answer tokens immediately. Skip to step 9.
Embed the question: call EmbeddingService with [question]. Returns float[1536].
Vector search: QdrantService.search(collection=projectId, vector=questionEmbedding, topK=15, filter={projectId:projectId}).
Metadata search: MetadataFilter.query(projectId, question) — extract layer keywords (controller/service/repository/entity) from question text using simple string matching. Run SELECT * FROM symbols WHERE project_id=? AND layer_type=ANY(?) LIMIT 10.
Merge: combine vector search results and metadata results. Deduplicate by chunk ID. Final score = 0.7 * vectorScore + 0.3 * (1.0 if in metadataResults else 0.0). Sort descending.
Select top 8 chunks by final score. If context.filePath provided in request, boost any chunk from that file by +0.2.
Build prompt using PromptBuilder (see Prompt Template below).
Call GPT-4o via LlmService with streaming=true. Pipe each token to SseEmitter. On completion: send final event { done:true, sources:[...] }.
Cache answer in Redis for 15 minutes.
